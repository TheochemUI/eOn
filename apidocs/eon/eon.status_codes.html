eon.status_codes¶

Process-search termination codes shared with the C++ client.

Mirror of client/StatusTypes.h. The integer values are wire format written to results.dat by eonclient, so they MUST match the C++ eonc::SaddleStatus enumerators byte-for-byte. The Python side exposes them as IntEnum members so server code can compare against SaddleStatus.Good instead of the magic literal 0, while still accepting raw ints transparently from parse_results.

Whenever client/StatusTypes.h adds, removes, or renumbers a code, update this module in the same commit and add a row to the label / slug tables.

eon.status_codes.saddle_status_label(code: int) → str[source]¶

Human-readable label for a saddle-search termination code.

Tolerant of unknown codes – returns a generic placeholder rather than raising, so a future C++ rev cannot cause an IndexError or KeyError to crash long-running akmc jobs.
