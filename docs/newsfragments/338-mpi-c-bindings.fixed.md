Port `MPIPot.cpp` and the MPI control flow in `ClientEON.cpp` off the removed MPI C++ bindings (`MPI::COMM_WORLD`, `MPI::INT`, `MPI::DOUBLE`, `MPI::Group`, ...). Every call uses the spec-compliant C bindings now (`MPI_Send`, `MPI_Recv`, `MPI_Iprobe`, `MPI_Comm_*`, `MPI_Group_*`, ...), so `-Dwith_mpi=true` builds against modern conda-forge MPICH 4.x and OpenMPI 5.x even though they ship without `mpicxx.h`. Wire format on the MPIPot Send/Recv channel is preserved by switching the local `icwd` buffer from `long[1024]` to `int[1024]` (the original code tagged the message `MPI::INT` while passing a long array, and any deployed MPI server already parses 1024 ints).
