Make the libtorch component lookup `required: false` and skip missing libraries (closes #304). Aligns with the CMake build's `find_package(Torch)` behaviour and unblocks Windows / cross-builds where `torch_global_deps` ships only a `.dll` (no `.lib` import library), as well as conda-forge `libtorch *cpu*` installs that strip `torch_cuda`. Combined with the `with_xtb` -> always-compile + dlopen change earlier in this PR (issue 1), the `xtb` fallback removal (issue 2), and the VLA -> `std::vector` swap inside the XTBPot rewrite (issue 3), all four meson cross-platform packaging items in #304 are now upstream.
